# UI Guideline

## Direction

The interface should feel close to Apple, Google Travel, Klook, and KKday:

- Minimal
- Modern
- Card based
- Mobile first
- Clear hierarchy
- Natural motion

## Mobile First

- Design for phones first.
- Minimum supported mobile width: 360px.
- Avoid layouts that require horizontal scrolling except intentional small controls.
- Important actions should be reachable with one thumb.

## Visual Style

- Use card-based sections for itinerary content.
- Prefer rounded XL corners.
- Use soft shadows, not heavy shadows.
- Glassmorphism can be used moderately, especially for summary cards or hero overlays.
- Keep spacing generous but not wasteful.

## Day Color Rules

- Day 1: Red
- Day 2: Green
- Day 3: Blue
- Day 4: Yellow
- Day 5: Purple

These colors should remain consistent across markers, cards, route accents, and day labels.

## Animation

- Animations should feel natural and helpful.
- Avoid overly flashy effects.
- Prefer subtle hover, focus, accordion, map movement, and route highlight transitions.

## Map and Timeline

- Map interactions should reinforce the itinerary.
- Timeline items should eventually sync with map FlyTo and marker highlight.
- Route lines should be readable without overwhelming the map.
